Getting Started with GitHub

Setting Up GitHub

Creating a GitHub Account

To get started with GitHub, you first need to create a GitHub account. This will allow you to host and manage your code repositories, collaborate with other developers, and use many other features of GitHub.

Installing and Configuring Git

You will also need to install Git on your local machine. Git is a version control system that lets you manage and keep track of your source code history. After installing Git, you need to configure it to connect to your GitHub account.

Creating a Repository

What is a Repository?

A repository is like a project folder where you store all your project files. It also includes all the version control history of your project.

Creating a New Repository

You can create a new repository on GitHub to start a new project. When creating a new repository, you can choose to initialize it with a README, .gitignore, or license.

Using GitHub Pages

What is GitHub Pages?

GitHub Pages is a static site hosting service that takes HTML, CSS, and JavaScript files straight from a repository on GitHub, optionally runs the files through a build process, and publishes a website.

Creating a GitHub Pages Site

To create a GitHub Pages site, you need to create a new repository (or use an existing one), push your site files to the repository, and then configure your repository settings to use GitHub Pages.

Creating and Merging Pull Requests

What is a Pull Request?

Pull requests let you tell others about changes you've pushed to a branch in a repository on GitHub. Once a pull request is opened, you can discuss and review the potential changes with collaborators and add follow-up commits before your changes are merged into the base branch.

Creating a Pull Request

To create a pull request, you need to have made your changes in a separate branch. This branch should be based on the branch you wish to merge your changes into.

Merging a Pull Request

Once your pull request has been reviewed and the branch passes your tests, you can send your pull request. Anyone with push access to the repository can complete the merge.

Managing GitHub Issues

What are GitHub Issues?

Issues are a great way to keep track of tasks, enhancements, and bugs for your projects. They’re kind of like email—except they can be shared and discussed with the rest of your team.

Creating an Issue

Anyone with read access to a repository can create an issue. If you have write access to a repository, you can assign the issue to someone else.

Closing an Issue

You can close an issue directly from a commit if you include certain keywords in your commit message.
